## 1. Independent Variables (IVs)

### IV1: Voice Cloning System (VC tool)
- **Type:** Categorical, between-stimuli (fully crossed)
- **Levels:** N VC systems (recommended N = 4–5)
- **Candidate tools** (based on ClonEval + availability):
  1. **XTTS-v2** (Coqui) — best performer in ClonEval
  2. **OuteTTS** — worst performer in ClonEval (provides range)
  3. **WhisperSpeech** — mid-range open-source
  4. **SpeechT5** (Microsoft) — mid-range, different architecture
  5. **VALL-E X** (Microsoft) — cross-lingual, different paradigm
- **Selection criteria:** All must be (a) open-source/freely available, (b) capable of processing the same input audio, (c) covering a range of quality levels (ensures variance in the IV)

### IV2: Lipsync System (Lipsync tool)
- **Type:** Categorical, between-stimuli (fully crossed)
- **Levels:** M lipsync systems (recommended M = 3–4)
- **Candidate tools** (based on THEval + AV-Deepfake1M++ + availability):
  1. **Wav2Lip** — most widely used baseline
  2. **SadTalker** — popular, generates head motion + lips
  3. **VideoReTalking** — high-quality visual output
  4. **MuseTalk** — recent, real-time capable
- **Selection criteria:** All must (a) accept a source image/video + audio as input, (b) be open-source, (c) cover a range from baseline (Wav2Lip) to SOTA

### IV3: Speech Condition (Emotion)
- **Type:** Categorical, within-stimuli
- **Levels:** 2 levels
  1. **Neutral** — emotionally flat speech
  2. **Emotional** — expressive speech (anger, happiness, sadness — pooled or tested separately depending on power)
- **Rationale:** ClonEval found performance drops for emotional speech. Starting with neutral vs. emotional (pooled) is most powerful; individual emotions can be exploratory.

### Design summary
- **Factorial design:** N (VC) × M (lipsync) × 2 (speech condition)
- **With N=4, M=4:** 4 × 4 × 2 = **32 conditions**
- **With N=5, M=3:** 5 × 3 × 2 = **30 conditions**
- **Recommended: 4 × 4 × 2 = 32 conditions** (balanced, manageable)

## 3. Design Type

**Mixed factorial design:**
- **Between-stimuli:** VC tool (IV1) × Lipsync tool (IV2) — each stimulus is generated by one specific VC + lipsync combination
- **Within-stimuli:** Speech condition (IV3) — the same source identity/text appears in both neutral and emotional conditions
- **Within-subjects:** All human raters evaluate all conditions (complete design)

**Justification:**
- Within-subjects for raters reduces individual differences in quality perception and requires fewer participants
- Fully crossed VC × lipsync is essential for testing the interaction hypothesis (H3)
- Within-stimuli for emotion allows direct paired comparison (H4)

## 4. Stimuli Construction

### Source material
- **Source dataset:** MEAD (recommended) — contains neutral + 8 emotions from 48 actors, high resolution, publicly available. Alternative: CREMA-D or RAVDESS.
- **Select:** K source identities × 2 speech conditions (neutral + emotional) × S sentences per condition
- **Recommended:** K = 5 identities (2 male, 2 female, 1 diverse), S = 2 sentences per condition = **20 source clips** (5 × 2 conditions × 2 sentences)

### Generation pipeline
For each source clip:
1. Extract audio → pass through each VC system → N cloned audio tracks
2. For each cloned audio + source identity frame → pass through each lipsync system → N × M generated videos
3. Total stimuli: **20 source clips × 4 VC × 4 lipsync = 640 generated videos** (for N=M=4)

### Ground truth
- Include **original unmodified videos** as a reference condition (upper bound)
- Include **mismatched audio** (wrong identity) as a lower-bound/attention-check condition

## 5. Confounding Variables to Control

| Confound | Control strategy |
|----------|-----------------|
| **Source identity** | Use the same identities across all conditions; counterbalance identity assignment |
| **Source audio quality** | All source clips from the same dataset (MEAD), same recording conditions |
| **Text content** | Same sentences used across all VC × lipsync × emotion conditions (within each identity) |
| **Video resolution/format** | Standardize all output to the same resolution, frame rate, and codec before evaluation |
| **Video duration** | Standardize clip length (5–10 seconds recommended) |
| **Presentation order** | Randomize stimulus order per participant (Latin square or full randomization) |
| **Rater fatigue** | Limit session to ~30 min; include breaks; randomize |
| **Language familiarity** | Use English-only stimuli (or control for language as a covariate) |

## 6. Statistical Tests

### Primary analyses

| Hypothesis | Test | Design | DV |
|-----------|------|--------|-----|
| H1 (VC main effect) | Two-way mixed ANOVA or linear mixed model (LMM) | Factor: VC tool (4 levels) | Each human MOS + each computational metric |
| H2 (Lipsync main effect) | Same ANOVA / LMM | Factor: Lipsync tool (4 levels) | Same |
| H3 (VC × Lipsync interaction) | Same ANOVA / LMM | VC × Lipsync interaction term | Same — **primary test** |
| H4 (Emotion degradation) | Paired t-test or within-subjects ANOVA | Neutral vs. Emotional (within) | Same |
| H5 (Emotion × Tool interaction) | Three-way mixed ANOVA / LMM | VC × Lipsync × Emotion | Same |
| H6 (Metric validity) | Spearman rank correlation + bootstrap CI | Correlate each metric with average human MOS across 32 conditions | ρ per metric |

### Recommended approach: Linear Mixed Model (LMM)

**Formula:**
```
MOS ~ VC_tool * Lipsync_tool * Speech_condition + (1|participant) + (1|source_identity)
```

**Why LMM over repeated-measures ANOVA:**
- Handles the nested structure (ratings nested within participants, stimuli nested within source identities)
- Handles unbalanced data (missing ratings, participant dropout)
- Can model random slopes if needed
- Standard in recent speech/audio evaluation literature

**Software:** R (`lme4` package) or Python (`statsmodels` / `pymer4`)

### Multiple comparisons correction
- **Method:** Benjamini-Hochberg FDR (controls false discovery rate)
- **Applied to:** Post-hoc pairwise comparisons within significant main effects (e.g., which VC tool pairs differ)
- **Why BH over Bonferroni:** With 32 conditions and multiple DVs, Bonferroni is too conservative. BH provides better power while controlling FDR at α = 0.05.

## 7. Effect Size Measures

| Test | Effect size | Interpretation benchmarks |
|------|-----------|--------------------------|
| ANOVA main effects | **η² (eta-squared)** or **partial η²** | Small: 0.01, Medium: 0.06, Large: 0.14 |
| Pairwise comparisons | **Cohen's d** | Small: 0.2, Medium: 0.5, Large: 0.8 |
| Metric-human correlation | **Spearman ρ** | Weak: <0.3, Moderate: 0.3–0.7, Strong: >0.7 |
| LMM | **Marginal R²** (fixed effects) + **Conditional R²** (fixed + random) | Report both |

### Stimuli per participant
- Each participant rates all 32 conditions (within-subjects)

- **Recommended:** Show 2 identities per participant (counterbalanced), = 64 ratings ≈ 25 min session (at ~20s per rating including viewing + response)

## 9. Exclusion Criteria

### For participants
1. **Attention check failures:** Exclude participants who rate the clearly-bad attention check videos (mismatched audio) as high quality (≥4/5 on overall quality) more than 50% of the time
2. **Response time:** Exclude ratings with response time < 1 second (random clicking) or > 60 seconds (distracted)
3. **Flat responding:** Exclude participants who give the same rating to > 90% of stimuli (no discrimination)
4. **Non-completion:** Exclude participants who complete < 80% of stimuli

### For stimuli
1. **Generation failure:** Exclude stimuli where the VC or lipsync system produces corrupted/empty output (document count and system)
2. **Gross artifacts:** If a system crashes on a specific input, exclude that cell and note it (do NOT replace with interpolated data)

### Pre-registered decision: Intent-to-treat
- Report results both with and without exclusions
- Primary analysis: with exclusions applied
- Sensitivity analysis: all data included
